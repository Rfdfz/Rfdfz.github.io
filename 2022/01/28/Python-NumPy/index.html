Python - NumPy

发表于 2022-01-28 更新于 2022-03-18 分类于 数据分析
本文字数： 5.7k 阅读时长 ≈ 5 分钟

NumPy是Python的一种开源的数值计算扩展。这种工具可用来储存和处理大型矩阵，比Python自身的嵌套列表（nested list structure）结构要高效得多。NumPy（Numeric Python）提供了许多高级的数值变成工具。NumPy的一个重要特性是它的数组计算，是我们做数据分析必不可少的一个包

生成全0数组：a = np.zeros(SIZE, dtype=Type)SIZE为数组大小，Type为数组元素类型（数组元素默认为浮点数）

生成全1数组：a = np.ones(SIZE, dtype=Type)SIZE为数组大小，Type为数组元素类型（数组元素默认为浮点数）

可以使用fill方法将数组设为指定值：a.fill(Num)Num为填充数值（若参数类型与原来不同，以原来的为准）

类型强制转换：a = a.astype(Type)Type为类型

生成整数数列：a = np.arange(min, max[, step])min为起始值，max为终点值（不包括max），step为步长值

生成等差数列：a = np.linspace(min, max, num)min为起始值，max为终点值（包括max），num为元素个数

生成随机数数组：

a = np.random.rand(num)num为随机数个数（默认取值范围为[0,1]的浮点数）

a = np.random.randn(num)num为随机数个数（服从标准正态分布的随机浮点数）

a = np.random.randint(min, max, num)min为起始值，max为终点值，num为随机数个数

切片是引用

切片在内存中使用的是引用机制

引用机制意味着，Python并没有为b分配新的空间来存储它的值，而是让b指向了a所分配的内存空间，因此，改变b会改变a的值：

而这种现象不会再列表中出现，列表的切片是在内存开辟了一片新的空间用于存储一个新的列表：

引用机制的好处在于，对于很大的数组，不用大量复制多余的值，节约了空间

引用机制的缺点在于，可能出现改变一个值改变另外一个值的情况

一个解决方法是使用copy()方法产生一个复制，这个复制会申请新的内存：
